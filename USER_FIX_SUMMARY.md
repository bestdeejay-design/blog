# 👥 Управление пользователями - Исправление ошибок

## ✅ Что было исправлено

### Проблема:
При создании или редактировании пользователя появлялась ошибка **"Ошибка при создании пользователя"**

### Причина:
Функция `createUser` создавала запись только в таблице `user_profiles`, но не создавала пользователя в системе аутентификации Supabase (auth.users).

### Решение:
1. **Обновлена функция `createUser`** в [lib/db.ts](file:///Users/admin/Documents/GitHub/blog/lib/db.ts)
   - Теперь создает пользователя в Supabase Auth
   - Затем создает профиль в user_profiles
   - Добавлена обработка ошибок с подробными сообщениями

2. **Добавлена функция `updateUser`** в [lib/db.ts](file:///Users/admin/Documents/GitHub/blog/lib/db.ts)
   - Позволяет редактировать пользователей
   - Обновляет данные в user_profiles
   - При необходимости обновляет пароль

3. **Создан API endpoint `/api/users/update`**
   - PUT запрос для обновления пользователей
   - Валидация данных
   - Проверка уникальности username

4. **Обновлен интерфейс в Dashboard**
   - Добавлена кнопка редактирования (✏️) для каждого пользователя
   - Добавлено модальное окно редактирования
   - Улучшена обработка ошибок

## 🚀 Как создать пользователя

### Через админ-панель:

1. Войдите в систему (admin@blog.local / Admin@2026Secure!)
2. Перейдите на вкладку **"👥 Пользователи"**
3. Нажмите **"➕ Добавить пользователя"**
4. Заполните форму:
   - **Логин** - уникальное имя (обязательно)
   - **Пароль** - мин. 6 символов (обязательно)
   - **Имя** - необязательно
   - **Роль** - редактор/админ/супер-админ (обязательно)
5. Нажмите **"✅ Создать пользователя"**

### Результат:
- Пользователь создается в Supabase Auth
- Создается профиль в user_profiles
- Страница перезагружается с новым списком пользователей

## ✏️ Как редактировать пользователя

**Только супер-админ может редактировать!**

1. На вкладке **"Пользователи"** найдите нужного пользователя
2. Нажмите на иконку **✏️** рядом с именем
3. Откроется окно редактирования
4. Измените нужные поля:
   - **Логин** - новое имя пользователя
   - **Пароль** - оставьте пустым, чтобы не менять
   - **Имя** - полное имя
   - **Роль** - выберите новую роль
5. Нажмите **"💾 Сохранить изменения"**

## 🐛 Возможные ошибки и решения

### "Пользователь с таким именем уже существует"
**Причина:** Username должен быть уникальным

**Решение:**
- Используйте другое имя
- Или отредактируйте существующего пользователя

### "Пароль должен быть не менее 6 символов"
**Причина:** Короткий пароль

**Решение:** Используйте пароль длиннее 6 символов

### "Auth user creation error: User already registered"
**Причина:** Email уже существует в Supabase Auth

**Решение:**
1. Проверьте список пользователей
2. Используйте другой username/email
3. Или удалите существующего пользователя в Supabase Dashboard
