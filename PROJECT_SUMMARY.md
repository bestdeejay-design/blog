# 📋 РЕЗЮМЕ ПРОЕКТА

## Что было создано

Полноценная **Multi-Channel News Management System** — система управления новостями для нескольких сайтов с единой админ-панелью.

## 🚀 Как использовать

### 1️⃣ Настроить Supabase
- Создать проект
- Выполнить `supabase-schema.sql`
- Получить API ключи

### 2️⃣ Настроить проект
```bash
git clone https://github.com/bestdeejay-design/blog.git
cd blog
npm install
cp .env.local.example .env.local
# Заполнить .env.local значениями из Supabase
```

### 3️⃣ Запустить локально
```bash
npm run dev
# http://localhost:3000
```

### 4️⃣ Зарегистрироваться
- Войти через `/login`
- В Supabase изменить роль на `super_admin`

### 5️⃣ Создать канал и новость
- Через админ-панель

### 6️⃣ Интегрировать на сайты
```tsx
import NewsFeed from '@/components/news-feed'

<NewsFeed baseUrl="https://your-app.com" limit={5} />
```

### 7️⃣ Задеплоить
```bash
vercel
```

## 📊 База данных — схема

```
channels (каналы/сайты)
├── id, name, slug, url, description
└── is_active, timestamps

user_profiles (редакторы)
├── id (FK to auth.users), email, full_name
├── role (super_admin/admin/editor)
└── is_active, timestamps

channel_editors (связь редакторов с каналами)
├── channel_id (FK), user_id (FK)
├── permissions (JSONB): {can_create, can_edit, can_delete, can_publish}
└── granted_by, granted_at

news (новости)
├── id, channel_id (FK), author_id (FK)
├── title, slug, content, excerpt, image_url
├── status (draft/published/hidden/archived)
├── views_count, published_at
└── timestamps

news_channels (мульти-канал публикации)
├── news_id (FK), channel_id (FK)
└── published_at
```
