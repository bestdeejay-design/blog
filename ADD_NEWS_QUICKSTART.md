## 🚀 Пошаговая инструкция

### Шаг 1: Настройка базы данных (ОБЯЗАТЕЛЬНО!)

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Выберите ваш проект
3. Перейдите в **SQL Editor**
4. Скопируйте содержимое файла: `supabase/migrations/20260321_create_database_schema.sql`
5. Нажмите **Run** для выполнения

**Что будет создано:**
- Таблица `news` - хранение новостей
- Таблица `news_channels` - публикация в несколько каналов
- Индексы для производительности
- RLS политики для безопасности

### Шаг 2: Создание канала (если нет каналов)

Перед добавлением новости нужен хотя бы один канал:

1. Откройте http://localhost:3000/login
2. Войдите как супер-админ:
   - Email: `admin@blog.local`
   - Пароль: `Admin@2026Secure!`
3. Перейдите на вкладку **"📺 Каналы"**
4. Нажмите **"➕ Добавить канал"**
5. Заполните:
   - Название: `Main News`
   - Ключ (slug): `main-news`
6. Нажмите **"✅ Создать канал"**

### Шаг 3: Добавление новости

1. Перейдите на вкладку **"📰 Новости"**
2. Нажмите **"➕ Добавить новость"**
3. Заполните форму:
   - **Заголовок**: `Моя первая новость`
   - **Текст новости**: `Текст новости...`
   - **Канал**: выберите из списка
   - **Статус**: 
     - `Черновик` - видно только вам
     - `Опубликовано` - доступно публично
4. Нажмите **"✅ Создать новость"**

### Шаг 4: Проверка результата

**Способ 1: Через консоль браузера**
- Откройте DevTools (F12)
- Посмотрите логи:

## 🔧 Технические детали

### API Endpoint
- **URL**: `POST /api/news/create`
- **Content-Type**: `multipart/form-data`
- **Параметры**:
  - `title` (string, обязательно) - заголовок
  - `content` (string, обязательно) - текст новости
  - `channel_id` (UUID, обязательно) - ID канала
  - `status` (string, опционально) - "draft" или "published"

### Пример curl запроса:
```bash
curl -X POST http://localhost:3000/api/news/create \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "title=Тестовая новость" \
  -d "content=Текст тестовой новости" \
  -d "channel_id=UUID_КАНАЛА" \
  -d "status=published"
```

### Структура таблицы news:
```sql
CREATE TABLE news (
  id UUID PRIMARY KEY,
  channel_id UUID REFERENCES channels(id),
  author_id UUID REFERENCES user_profiles(id),
  title VARCHAR(500) NOT NULL,
  content TEXT NOT NULL,
  status VARCHAR(50) DEFAULT 'draft',
  published_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 🐛 Решение проблем

### Ошибка: "relation 'news' does not exist"

**Проблема:** Таблица не создана в БД

**Решение:**
1. Выполните миграцию из Шага 1
2. Проверьте в Supabase Dashboard → Table editor

### Ошибка: "Заголовок и канал обязательны"

**Проблема:** Не заполнены обязательные поля

**Решение:** Убедитесь, что заполнены "Заголовок" и выбран "Канал"

### Ошибка: "insert or update on table 'news' violates foreign key constraint"

**Проблема:** Несуществующий ID канала

**Решение:**
1. Создайте канал сначала
2. Проверьте список каналов в вкладке "Каналы"

### Пустой список каналов в форме

**Проблема:** Нет созданных каналов

**Решение:** Создайте канал (см. Шаг 2)
